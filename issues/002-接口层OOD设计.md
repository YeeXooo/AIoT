# 接口/API 层 OOD 设计

> 状态：待开始
> 前置：`docs/ood_domain.md` ✅ / `docs/ood_application.md` ✅ / `docs/ood_infrastructure.md` ⬜

## 描述

为车载安全监测系统完成接口/API 层的契约级 OOD 设计文档。定义六个应用服务的 REST API 契约、MQTT 主题路由、WebSocket/SparkRTC 信令、ArkTS 前端对接契约及 API 安全策略。

## 技术栈

- 后端：Java Spring Boot
- 前端：ArkTS（HarmonyOS）
- 设备通信：MQTT（华为云 IoTDA）
- 音视频：SparkRTC
- 认证：JWT/OAuth2

## 产出要求

### 1. REST API 契约（OpenAPI 3.0 风格）
六个应用服务的全部端点：
- **S1 RiskMonitoringService**：流式判定会话管理、历史风险查询
- **S2 InterventionService**：干预状态查询、驾驶员覆盖上报
- **S3 RemoteGuardianshipService**：家属权限查询/管理、状态订阅、音视频对讲请求、远程车窗控制
- **S4 FleetManagementService**：看板查询、钻取查询、报告生成/下载、绩效预警订阅
- **S5 EmergencyRescueService**：SOS 确认、救援授权管理、家属手动救援触发
- **S6 OTAManagementService**：升级任务创建/查询、回滚指令下发、升级进度查询

每个端点含：路径、HTTP 方法、请求体/查询参数、响应体、HTTP 状态码、认证头。

### 2. MQTT 主题设计
- 完整 Topic 路由表（边缘→云感知上报、云→边缘指令下发、云端推送→家属 APP）
- QoS 等级定义
- Payload 格式（JSON Schema）

### 3. WebSocket/SparkRTC 集成
- 家属 APP 音视频对讲、远程视频监控的 WebSocket 信令协议
- SparkRTC 房间管理接口

### 4. ArkTS 前端对接契约
- 家属 APP 调用的全部后端接口清单与 DTO 定义
- 车队大屏的看板数据订阅模型
- HMI（车机端）的本地查询接口

### 5. API 安全设计
- 认证方案（JWT/OAuth2）
- 接口限流策略
- MQTT 设备鉴权（X.509 证书 / Token）
- 敏感数据加密传输策略
